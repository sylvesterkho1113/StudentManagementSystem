# 🧑🏻‍🎓 Student Management System using Semantic Web Technology

A Student Management System leveraging semantic web technologies including XML, RDF, and SPARQL. This project is implemented fully in Python and showcases how semantic web standards can be applied in real-world educational data management.

## 🛠️ Features:
1. Manage student and subject records using XML.
2. Store and update student grades.
3. Calculate CGPA based on credit hours and grading systems.
4. Recommend subjects using SPARQL based on programme and prerequisite status.
5. Generate reports with FPDF.
6. Use of RDF graphs and SPARQL to query and reason with semantic data.

## 📂 Project Structure

```
Student-Management-System/
├── main.py                 - Main entry point of the system. Provides menu to access student, subject, grading, and reporting modules.
├── student_module.py       - Handles student-related operations (add, display, delete). Uses XML for data storage.
├── subject_module.py       - Manages subjects, including prerequisite checks and filtering by programme.
├── grading_module.py       - Allows grade entry, GPA calculation, CGPA update, and eligibility filtering.
├── rdf_utils.py            - Turn XML file into RDF graph, and perform searching using SPARQL.
├── grading_share_utils.py  - Contains functions that are used by both rdf_utils.py and grading_module.py
├── data/                   - XML and RDF files generated by system will be stored in this folder.
│   ├── grades.rdf
│   ├── grades.xml
│   ├── subjects.rdf
│   ├── subjects.xml
│   ├── students.rdf
│   └── students.xml
├── report/                 - Student report, subject report, and student result slip generated by system will be stored in this folder.
│   ├── resultslip_S001.pdf
│   └── resultslip_S002.pdf
└── README.md
```
